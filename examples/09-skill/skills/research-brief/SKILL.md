# Research Brief Skill

Use this skill when the user asks for technical research, comparison, investigation notes, or a decision brief.

## Workflow

1. Restate the research question in one sentence.
2. Separate confirmed facts from assumptions.
3. Compare options by tradeoff, not by feature lists alone.
4. Call out uncertainty and what would change the recommendation.
5. End with a practical recommendation and next steps.

## References

If the brief needs a more formal structure, read:

- `references/report-template.md`

## Output Format

Use these sections:

1. Question
2. Key Findings
3. Tradeoffs
4. Recommendation
5. Next Steps
