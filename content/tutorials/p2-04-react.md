## 本章定位

有了工具以后，Agent 仍然只能做“单步决策”。这一章要让读者看到，多轮观察驱动的执行模式为什么比单轮调用更强。

## 本章目标

- 理解 ReAct 的核心思想
- 基于现有工具系统实现一个多轮执行器
- 讲清楚 ReAct 的适用边界和限制

## 建议内容结构

### 1. 为什么单轮决策不够

先用一个需要多次查找和修正的任务引出问题。

### 2. ReAct 的最小执行模型

建议用一张图解释：

- 当前目标
- 决定动作
- 获得观察
- 更新状态
- 再决定下一步

### 3. 在代码里实现 ReAct

在第二章最小 Loop 的基础上，增加：

- 多轮工具调用
- 每轮 observation 回填
- 轮次限制
- 工具失败后的分支

### 4. 一个真实任务示例

例如：

- “先找到安装文档，再提取启动命令”
- “先搜索，再读取，再总结”

### 5. 什么时候不适合用 ReAct

例如长链路任务、明显需要先拆解的任务、代价高的搜索型任务。

## 建议代码产出

- `react-agent.ts`
- `observation-log.ts`
- 一个多轮任务示例
